> 版本：v1.0 需求基线（2026-07-20，经 12 轮 grilling 拷问收敛确认）
> 工作名：AirLine（正式命名上架前另定）
> 平台：iOS 原生 App
>
> **修订 2026-07-21（定位调整）**：v1 以独立开发者个人作品的形式交付，暂不引入付费开发者账号（$99/年）。

> 产品与代码设计仍按可上架标准执行（如不使用航司 logo）。

---

## 1. 一句话定位

以全球真实航线模拟飞行为外壳的 Forest 式专注 App：每次专注 = 飞一段真实航线，落地点亮城市，从落地城市继续下一程；用"常旅客计划"（里程、舱位、护照）驱动长期留存。

## 2. 目标与原则

- **目标**：上架 App Store 的正式产品；v1 无账号、无传统后端，数据本地存储 + iCloud 自动同步。
- **真实感原则**：航线、航司、航程时间、地理连通性全部来自真实世界数据。压缩只发生在"专注时钟"上，世界观里用户飞的始终是那趟真实航班。
- **仪式感原则**：登机牌、值机、备降、调机、舱位、护照——一切机制使用真实航空术语，用户零学习成本。
- **克制原则**：v1 不做社交、不做强惩罚、不做多语言，先跑通核心循环。

## 3. 核心循环

```
选目的地（登机牌卡片墙）→ 值机（生成登机牌）→ 专注飞行（航图模式 + 灵动岛）
→ 落地结算（点亮城市 / 里程入账 / 护照盖章 / 城市小传）→ 从落地城市开始下一程
```

### 3.1 时长映射（压缩映射）

- **基准压缩比约 5:1**：`专注时长 ≈ 真实航程分钟 ÷ 5`，再吸附到友好档位。
- 友好档位（分钟）：`15, 20, 25, 30, 35, 40, 45, 50, 60, 75, 90, 120, 150, 180`（吸附至最近档位）。
- 实际比例因此在 4~6 倍间自然浮动（30 分钟专注 ≈ 2~3 小时真实航程），不僵硬、不完全固定——这是刻意设计。
- **最低专注门槛 15 分钟**：超短途航线（真实 <75 分钟）一律映射为 15 分钟，避免"伪专注"。
- 登机牌与飞行界面同时展示两个数字：**真实航程时间**（世界观）与**专注时长**（时钟）。
- 映射示例：

| 真实航线 | 真实航程 | 专注时长 |
|---|---|---|
| 上海虹桥 → 北京首都 | 约 2h15m | 25 min |
| 北京 → 东京羽田 | 约 3h25m | 40 min |
| 北京 → 新加坡 | 约 6h20m | 75 min |
| 北京 → 纽约肯尼迪 | 约 13h30m | 150 min（可接力） |

### 3.2 专注判定与失败（Forest 式）

- **锁屏/息屏不算破戒**（灵动岛倒计时本来就是锁屏看的）。
- **离开 App 超过 60 秒宽限期 → 判定"备降"**（技术上：切后台时记录时间戳，回前台结算离开时长；iOS 无法也无需后台监控其他 App）。
- 备降后果（短途单段航班）：目的地**不点亮**、里程**不入账**；但**不倒扣**任何存量进度，用户仍停留在出发城市。叙事用"备降"而非"坠机"，符合航空真实感且不过度负面。
- Screen Time API 强制屏蔽其他 App：**v1 不做**，留作后续版本的"头等舱功能"。

### 3.3 连通性规则

- 首次启动：输入乘客姓名 + 自选**主场机场**（默认推荐附近大型枢纽）。
- 此后**严格图连通**：只能飞当前所在城市的**真实直飞航线**。想点亮巴黎，就要规划一条真实可行的路线飞过去——这是长期留存的核心乐趣。
- 已飞过的航线可重复飞：城市只点亮一次，里程照常入账。
- **调机兜底**：若当前机场无可选航线（含解锁等级限制导致），可免费"调机"回上一个途经枢纽；调机不计里程、不点亮城市。

### 3.4 长航线接力（v1 必做）

- **解锁条件**：累计里程 ≥ 5,000 km（与"优选经济"升舱绑定，门槛刻意低，三四次短途即达——保留"解锁跨洋能力"的成就感又不挡新手太久）。
- **适用范围**：压缩后专注时长 > 60 分钟的航线可选择接力模式（也可一次坐完）。
- **规则**：
  1. 值机一次生成登机牌；之后每次专注**自选本段时长**（最短 25 分钟，末段剩多少飞多少），飞机沿大圆航线按比例推进，剩余时长实时可见。
  2. **同时只能有一条未完成旅程**（人在太平洋上空，没法从别的城市出发）；想飞别的须先飞完或主动放弃。
  3. 某段破戒（离开 >60s）：**只作废当前段**（该段无里程），已完成段保留在检查点，随时续飞。
  4. **主动放弃整条旅程 = 返航**：已完成段的里程保留，目的地不点亮，人回到出发城市。

## 4. 目的地选择：登机牌卡片墙

- 身处某城市时进入选择界面，展示当前城市可直飞的**若干张登机牌卡片**（建议 8~12 张）。
- 候选集生成策略：优先多样性（未点亮城市 > 已点亮城市），**混入 1~3 张锁定卡片**制造升级欲望。
- 可按**专注时长档**筛选（15~30 / 30~60 / 60+ 分钟 / 接力航线）。
- **卡片正面**：目的地城市名 + IATA 三字码 + 国家 + 真实航程 + 对应专注时长 + 承运航司。
- **卡片展开/翻面**：目的地**城市小传**（见 §6），作为选择依据——小传随 App 内置打包，离线可读。
- **锁定卡片**：明示解锁条件与当前进度（如"公务舱可解锁 · 还差 8,420 km"）。
- 选定 → 值机动画 → 生成登机牌 → 开始专注。

## 5. 登机牌

### 5.1 字段（完全按真实登机牌版式）

| 字段 | 来源 |
|---|---|
| 乘客姓名 | 首次启动输入（本地存储） |
| 航司名称 + 二字码 | 航线数据中的真实承运航司（如 CA / Air China） |
| 航班号 | **生成式**：真实二字码 + 合理数字（如 CA1831），按航线稳定可复现 |
| 舱位等级 | 用户当前舱位（Y/W/C/F 舱位码） |
| 起降城市 + IATA | 真实数据 |
| 起飞/到达当地时间 | 值机时刻即起飞；到达 = 起飞 + 真实航程（按数据集时区换算） |
| 登机口 / 座位号 | 生成（座位区间随舱位：F 头等 1~3 排，C 公务 4~12 排，W 优选 31~40 排，Y 经济 41~60 排） |
| 真实航程 + 专注时长 | 见 §3.1 |
| 条形码/二维码 | 装饰性生成 |

### 5.2 法律约束（已定决策）

- **文字性使用真实航司名称/代码**（事实性描述，低风险）；**不使用任何航司 logo**（商标侵权风险明确）。
- 自研一套高级登机牌视觉体系，用尾舵色带/字母组合传达航司识别度。
- 不接入付费时刻表 API；生成式航班号与真实观感无差别、零成本。

## 6. 城市小传（构建期全量生成，客户端内置）

### 6.1 架构

- **构建期生成、App 内置**：全部 3,691 个机场的小传写入 `AirLine/Resources/city_bios.json`（约 1.6 MB），客户端零联网、零密钥、零运行时成本。
- **同城副机场**：与同城主机场相距 <80 km 的副场自动复用主场小传（见 `tools/make_bio_worklist.py` / `tools/build_bios.py`）。
- **源稿管理**：`data/bios/*.json` 分批存放；`python3 tools/build_bios.py` 合并校验后打包。
- **不再使用**：Cloudflare Worker / 运行时 LLM 代理（已废弃）。

### 6.2 风格规范（从用户提供的 12 个范例提炼）

**结构**：`城市中文名(英文名)` + `【8~20 字高张力标签】` + `120~220 字正文`

**标签三种手法**：
- 反差对仗：「通往最保守圣城的最开放门户」
- 身份重定义：「被火车拉来的东方小巴黎」
- 地理意象：「北冰洋上的不冻港」

**正文三段式**：
1. **定位句**：行政地位/规模一句锚定（"俄罗斯鄂木斯克州首府""丹麦第二大城市"）。
2. **核心叙事**：必须建立在一个"反差"上——历史辉煌 vs 今日现实 / 名气错位 / 刻板印象反转 / 平静 vs 爆发。
3. **硬细节收尾**：1~2 个具体可验证的事实（"1222 年建校""全球唯一核动力破冰船舰队""圆桌是都铎伪造手办"）。

**语气红线**：信息密度优先；幽默克制、点到为止；可用引号玩梗（"尔滨""真·波特兰"）；精准且有意思，**不刻意毒舌**、不空洞抒情、不堆感叹号。

## 7. 成长体系（常旅客模型）

- **里程 = 经验值**：专注成功后按航线**真实公里数**入账（接力按已完成段比例入账）。
- **舱位 = 等级**，影响仪式感与航线解锁权限，不破坏计时公平性：

| 舱位 | 累计里程门槛* | 解锁内容 |
|---|---|---|
| 经济舱 Y | 0 | 真实航程 ≤3h（约 ≤2,500 km）的航线；基础登机牌 |
| 优选经济 W | 5,000 km | ≤6h（约 ≤5,000 km）航线；**长航线接力能力**；磨砂质感登机牌 |
| 公务舱 C | 25,000 km | 全部航线（含 6h+ 超远程直飞）；深色质感登机牌、专属航迹色 |
| 头等舱 F | 100,000 km | 金属黑卡登机牌、专属灵动岛细节、历史旅程回放动画 |

*门槛数值为草案，设计阶段可调。

- **护照系统**：每**首次落地一个新国家**盖一枚章，护照页可翻阅。
- v1 不做：机型收集、成就徽章、连续天数 streak（后续版本再议）。

## 8. 世界地图（本产品门面）

- **技术**：自绘矢量地图。Natural Earth（免费、公有领域）世界轮廓 GeoJSON → 构建期简化/转二进制打包 → SwiftUI Canvas 渲染（性能不足时升级 Metal）。零 API 成本、完全离线、美术 100% 可控。
- **明确不用**：谷歌地图（自定义暗色样式必须走 map ID，按 Dynamic Maps 计费，超 1 万次/月后 $7/千次，且路网/POI 信息对本场景冗余）；MapKit（免费但做不出目标风格）。
- **投影**：v1 等距圆柱投影（实现简单），美术阶段可评估更换。
- **视觉**：近黑深蓝底色、大陆轮廓微光描边；未点亮城市不可见或极暗；**已点亮城市 = 暖金辉光散点（带 bloom）**，点亮越多整图越亮；历史航迹为渐变大圆弧线。整体气质：高级、典雅、克制。
- **交互**：双指缩放、拖拽平移；点击城市光点弹出**城市卡**（小传 + 首次抵达日期 + 飞抵次数）。

## 9. 专注中界面与灵动岛

### 9.1 App 内主界面：航图模式

- 暗色世界地图的**局部视角**，飞机沿大圆航线弧线缓慢推进；已飞段高亮、未飞段虔诚暗淡。
- 上方：大号倒计时 + 航班信息条（航司/航班号/舱位/起降三字码）。
- 复用自绘地图资产，省电且气质统一。舷窗视角等花样留给高舱位后续版本。

### 9.2 灵动岛 / Live Activity（ActivityKit）

- **紧凑态**：左侧飞机图标 + 起降三字码（PEK→HND），右侧倒计时。
- **展开态**：航线弧线进度条 + 预计"落地"时刻 + 航班号。
- **锁屏**：横向登机牌式卡片。
- 倒计时用系统原生 timer 渲染（`Text(timerInterval:)`），无需后台刷新、纯本地可靠。
- 压缩映射后单段专注 ≤180 分钟，远低于 Live Activity 8 小时系统上限，无风险。
- 无灵动岛硬件的机型自动降级为锁屏 Live Activity，系统行为，无额外开发。

## 10. 看板（信息架构：三个 Tab）

1. **地图**：§8 的暗色世界地图（点亮城市、航迹、城市卡）。
2. **飞行**：当前旅程状态（接力进度/剩余里程）+ 发起新飞行（登机牌卡片墙）。
3. **我的**：
   - 统计总览：累计专注时长、累计里程、航段数、点亮城市数/国家数、当前舱位及升舱进度条；
   - 飞行日志：按时间倒序的**历史登机牌**列表（每次专注 = 一张可回看的登机牌）；
   - 护照页：国家盖章墙。
- v1 不做：社交/排行榜、周报月报、分享图导出。

## 11. 技术架构

| 项 | 决策 |
|---|---|
| 语言/UI | Swift + SwiftUI，最低 **iOS 17** |
| 持久化 | SwiftData + CloudKit（iCloud 自动同步，无账号体系） |
| 灵动岛 | ActivityKit + WidgetKit extension |
| 地图渲染 | SwiftUI Canvas（必要时 Metal），自绘 |
| 第三方依赖 | **零** |
| 航线数据 | GitHub `airline-route-data`（每周更新的全球客运航线 JSON，约 38,000 条，含承运航司/飞行分钟/公里数/机场坐标/时区），**构建期取快照**打包为精简二进制/SQLite，完全离线 |
| 地图数据 | Natural Earth 1:50m 陆地轮廓，构建期简化打包 |
| 城市中文名 | 构建期数据管线生成主要城市中英对照表，长尾回退英文名 |
| 城市小传 | 构建期全量生成，内置 `city_bios.json`（全离线） |
| 隐私 | 无账号；姓名仅本地 + 用户自己的 iCloud；App Store 隐私标签近乎全空 |

### 构建期数据管线（一次性脚本）

1. 拉取 `airline_routes.json` 快照 → 过滤客运机场 → 精简字段 → 生成 app 内置数据库；
2. Natural Earth GeoJSON → 抽稀简化 → 二进制轮廓数据；
3. 主要城市中文名映射表；
4. 全量机场城市小传内置打包（`city_bios.json`）。

**奥胡斯 (Aarhus)**
【维京海盗老巢与"彩虹光环"下的最年轻之城】丹麦第二大城市，日德兰半岛上的最大城市。在历史上，它曾是极其强悍的维京海盗建立的古老堡垒。因为这里坐落着庞大的奥胡斯大学，海量的学生涌入，这座古城其实是全丹麦人口结构最年轻的城市。在城市的天际线上，最瞩目的地标不是古堡，而是奥胡斯美术馆顶端的"彩色玻璃圆环"，人们可以在彩虹色的玻璃长廊里俯瞰整座城市。

**摩尔曼斯克 (Murmansk)**
【北冰洋上的不冻港】俄罗斯摩尔曼斯克州首府，北极圈内最大的城市。这里有着极其残酷的极昼极夜，在漫长的冬天，整座城市会陷入持续几十天的绝对黑暗。因为北大西洋暖流的眷顾，这里虽然地处极寒的北极圈，其港口的海水却终年不结冰。凭此它成为了俄罗斯北方舰队的大本营，也是全球唯一一支核动力破冰船舰队的老巢。

**巴彦浩特 (Bayanhot)**
【苍天大漠的边缘的引擎狂欢】内蒙古阿拉善盟的行政公署驻地。平时，这是一座极其安静、紧贴着腾格里沙漠边缘的塞外小城。但只要到了每年的"阿拉善英雄会"，全国各地的数百万越野车爱好者、顶级的改装SUV和重型越野车会疯狂涌入这片沙漠。震耳欲聋的引擎轰鸣和漫天的黄沙，这座小城骤变成如《疯狂的麦克斯》般的沙漠废土乌托邦。

**阿加德兹 (Agadez)**
【撒哈拉的泥土明珠与绝望的偷渡门户】西非国家尼日尔阿加德兹大区首府。这座沙漠古城在古代是阿加德兹苏丹国首府，极其繁华的跨撒哈拉贸易枢纽，拥有世界上最高的泥砖建筑——阿加德兹大清真寺，被誉为"沙漠大门"。但如今，这扇大门是撒南非洲通往欧洲偷渡路线上的中转站。无数试图穿越撒哈拉沙漠前往地中海的非法移民，会在这里挤上极其危险的皮卡车，驶向那片生死未卜的死亡沙海。

**波特兰 (Portland)**
【掷硬币输掉名气的"真·波特兰"】美国缅因州的最大城市。提到波特兰，全世界首先想到的都是美国西海岸常年下雨的俄勒冈州波特兰。当年俄勒冈州的建城者中，有一个人来自缅因州的波特兰，他和另一个来自波士顿的人通过抛硬币来决定新城市的名字，于是西海岸诞生了一座同名城市。在日后彻底抢走了缅因州这座遍布灯塔、被称为世界龙虾之都的原版波特兰的名气。

**哈尔滨 (Harbin)**
【被火车拉来的东方小巴黎】黑龙江省会。近年互联网语境中，它是搓澡、冰雕与极度热情的"尔滨"。但它的建城史充满异国情调——它其实是跟着俄罗斯人的"中东铁路"在荒野上拔地而起的。20世纪初，这里曾云集了几十个国家的领事馆和巨量外侨，不仅留下了满大街的巴洛克与拜占庭式建筑，更让它成为了中国最早接触西方古典乐的城市，被联合国教科文组织硬核加冕为亚洲唯一的"音乐之都"。

**吉达 (Jeddah)**
【通往最保守圣城的最开放门户】沙特阿拉伯的第二大城市与最大港口。在全球穆斯林的心中，它是神圣的"麦加门户"，每年无数朝圣者从这里下船或下飞机前往圣城。但它同时也是沙特这个极度保守的国家里，最开放、世俗、具有商业活力的城市。这里不仅矗立着世界上最高的海上喷泉（法赫德国王喷泉），还烂尾着那座曾试图击败哈利法塔的吉达塔。

**桃园 (Taoyuan)**
【台北入口与东南亚原乡】台湾省辖地级市。由于拥有全台最大的桃园国际机场，几乎所有飞往台湾北部的国际航班，降落地都写着桃园。除去让游客入境时误认为身在台北之外，它还是台湾产值极高的工业心脏。因为聚集了极其庞大的外籍劳工，这里的火车站周边形成了全台最硬核、最原汁原味的东南亚美食和生活圈。

**鄂木斯克 (Omsk)**
【无法逃离的白军旧都与互联网废土】俄罗斯鄂木斯克州首府。在俄国十月革命后的内战时期，这里曾短暂地做过反布尔什维克的"白军"首都，海军上将高尔察克曾带着大批沙俄帝国的黄金储备盘踞于此。但在现代俄罗斯互联网上，因为地处荒凉的西伯利亚腹地且交通规划奇葩，俄罗斯网民制作了海量"千万别试图离开鄂木斯克，因为你根本逃不出去"的废土风表情包，让它成为了赛博空间里的"寂静岭"。

**塔拉哈西 (Tallahassee)**
【最不像佛州的深南首府】美国佛罗里达州的首府。提到佛州，全世界想到的都是迈阿密的比基尼沙滩、奥兰多的迪士尼乐园。但这首府却气质迥异——它不靠海、没有沙滩，气候和文化都更接近北方的佐治亚州。它当选首府的原因也十分平实：当时佛州两个大城市彭萨科拉和圣奥古斯丁为了争首府打得不可开交，最后大家只能折中，在两地的地理中点选择了这块荒地。

**温彻斯特 (Winchester)**
【伦敦之前的英格兰绝对心脏】英国汉普郡古城。在今天，虽然如今人口不到五万人，但它才是英格兰为数不多的"千年古都"。作为古威塞克斯王国的首都，阿尔弗雷德大帝正是从这里出发，击退了维京海盗，奠定了英格兰的基础。在这座古城的大厅里，至今还挂着一张巨大的"亚瑟王圆桌"（虽然后来被证实是都铎王朝时期的伪造手办），依然散发着英格兰中世纪最纯正的权力幽香。

**帕多瓦 (Padova)**
【威尼斯背后的学术巨头】意大利北部古城。在旅游圈，它的光芒被近在咫尺的水城威尼斯彻底掩盖。但在人类文明的图谱上，它是颇具意义的学术灯塔。这里坐落着建于1222年的帕多瓦大学，现代科学之父伽利略曾在这里执教了18年。这里至今保留着世界上第一个"阶梯式木制解剖剧场"，近现代医学先驱们就是在这个极其幽闭的环形木屋里，在烛光下偷偷解剖尸体，开启了现代医学的曙光。

## 附录 B：事实调研记录（2026-07-20）

1. **航线数据**：GitHub `Jonty/airline-route-data`（及镜像 `mvanlaar/airline-route-data`）——每周自动更新的单文件 JSON，按机场 IATA 索引，含出港航线的目的地、承运航司（二字码+名称）、飞行分钟数、公里数、机场坐标与时区。OpenFlights 免费但 2014 年后停更，弃用。
2. **谷歌地图 iOS SDK**：不带 map ID 免费，但自定义暗色样式必须用 map ID → 按 Dynamic Maps SKU 计费（每月 1 万次地图加载免费，超出 $7/千次）。结论：弃用，自绘矢量地图。
3. **Live Activity**：最长活跃 8 小时（之后系统强制结束，锁屏最多再保留 4 小时）。压缩映射后单段专注 ≤3 小时，无冲突。
4. **iOS 后台限制**：普通 App 无法监控用户在用什么其他 App；Forest 式"离开即失败"通过前后台切换时间戳实现，技术简单可靠。Screen Time/FamilyControls 需特殊授权，v1 不碰。
